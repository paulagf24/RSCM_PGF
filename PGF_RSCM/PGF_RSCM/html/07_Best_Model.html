7. Optimización de hiperparámetros con validación cruzada en el modelo Random Forest¶

Los hiperparmetros son configuraciones externas al modelo de aprendizaje automtico que afectan su rendimiento pero que no son aprendidos automticamente durante el entrenamiento. A diferencia de los parmetros del modelo, que son aprendidos a partir de los datos durante el proceso de entrenamiento, los hiperparmetros deben ser establecidos antes de entrenar el modelo y no cambian durante el proceso de entrenamiento.

Los hiperparmetros son esenciales para optimizar el rendimiento de un modelo y ajustarlo mejor a un conjunto de datos especfico.

El proceso de ajustar y seleccionar los hiperparmetros ptimos se conoce como "optimizacin de hiperparmetros". Se ha empleado el mtodo de bsqueda aleatoria (Random Search) para mejorar la eficiencia del modelo, ya que a travs de la bsqueda en cuadrcula (Grid Search) resultaba un trabajo ms exhaustivo y minucioso.

In [8]:
# Hiperparmetros 
param_dist = {
    'n_estimators': [50, 100, 200],
    'max_depth': [None, 10, 20],
    'min_samples_split': [2, 5, 10],
    'min_samples_leaf': [1, 2, 4]
}

Descripcin de los hiperparmetros En un modelo de Random Forest, los parmetros especificados en el param_grid son hiperparmetros que se pueden ajustar para optimizar el rendimiento del modelo. Aqu est la interpretacin de estos parmetros:

  1. n_estimators:

    • Se trata del nmero de rboles en el bosque. En este caso se estn probando tres valores: 50, 100 y 200 rboles. Cunto mayor sea el nmero de rboles, ms robusto y generalizado puede ser el modelo, pero esto tambin puede hacer que aumente el coste computacional.

  2. max_depth:

    • Describe la profundidad mxima de cada rbol en el bosque. Se emplean tres valores: 'none', sin lmite de profundidad, 10 y 20. De esta forma, se controla la complejidad de cada rbol y, por lo tanto, el nivel de sobreajuste, en este caso, que se podra haber producido al obtener unas buenas mtricas durante el entrenamiento del modelo.

  3. min_samples_split:

    • Este hiperparmetro ensea el nmero mnimo de muestras requeridas para dividir un nodo interno. Se prueban tres valores: 2, 5 y 10. Cuanto mayor sea este valor, ms probable es que los rboles tengan nodos ms grandes y, por lo tanto, una mayor capacidad para sobreajustar.

  4. min_samples_leaf:

    • El nmero mnimo de muestras requeridas para estar en un nodo hoja, probando con valores 1, 2 y 4. Controla la cantidad mnima de muestras en una hoja, lo que puede influir en la suavidad del modelo.

Al realizar una bsqueda aleatoria con estos conjuntos de valores, se est explorando el espacio de hiperparmetros de manera ms eficiente en comparacin con una bsqueda de cuadrcula (grid search), que probara todas las combinaciones posibles. Esto es especialmente til, puesto que el espacio de bsqueda de datos puede llevar un coste de recursos computacionales elevados y que no resultan ilimitados.

Aplicación del Random Search con validación cruzada¶

In [25]:
random_search = RandomizedSearchCV(
    modelo_randomforest,   # Modelo de Random Forest 
    param_distributions=param_dist,  # Diccionario con los hiperparmetros y las distribuciones para muestrear
    n_iter=10,  # Nmero de combinaciones aleatorias a probar
    cv=5,  # Nmero de divisiones para la validacin cruzada 
    scoring=scorer,  # Mtrica de evaluacin a utilizar 
    random_state=42  # Semilla para reproducibilidad de los resultados
)

  1. modelo_randomforest: es el modelo de Random Forest empleado como estimador y definido previamente.

  2. param_distributions: es un diccionario que define el espacio de bsqueda de hiperparmetros. De esta forma, para cada hiperparmetro, proporcionas una distribucin desde la cual se muestrearn los valores durante la bsqueda aleatoria. Este diccionario contiene los hiperparmetros mencionados previamente: (n_estimators, max_depth, min_samples_split, min_samples_leaf) con las distribuciones asociadas.

  3. n_iter: es el nmero de combinaciones aleatorias de hiperparmetros que se probarn. En este caso, se estn probando 10 combinaciones aleatorias.

  4. cv: es el nmero de divisiones para la validacin cruzada. En este caso, se utiliza validacin cruzada estratificada con 5 divisiones (cv=5), lo que significa que el conjunto de datos se divide en 5 partes, y el modelo se entrena y evala 5 veces, utilizando cada parte como conjunto de prueba en una iteracin diferente.

  5. scoring: especifica la mtrica de evaluacin que se utiliza para evaluar la calidad de las combinaciones de hiperparmetros. En este caso se hace uso de 'custom_metrics', que parten de la sensibilidad y precisin as como de los resultados obtenidos en la matriz de confusin.

  6. random_state: es la semilla utilizada por el generador de nmeros aleatorios, y que, de esta forma, los resultados sean reproducibles.
